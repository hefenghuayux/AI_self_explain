# 阶段 07：暂停恢复、日志与审计

## 目标

补齐会话可恢复性、错误可观察性和完整操作审计。

## 范围

- 稳定阶段暂停和恢复。
- 处理中暂停限制。
- JSON 结构化日志。
- 请求关联 ID。
- 状态快照和错误查询。
- 日志轮转。

## 修改内容

- 只允许在 `WAIT_INITIAL_CHOICE`、`CAPTURING_INPUT`、`CONFIRMING_TEXT`、`WAIT_STUDENT_ACTION` 和 `SHOWING_FULL_SOLUTION` 暂停。
- `TRANSCRIBING` 和 `AI_EVALUATING` 期间等待调用结束后再处理暂停请求。
- 保存 `pausedFromStage`，恢复时保持轮次、计数和历史数据。
- 为请求、会话、外部调用和状态事件建立关联标识。
- 增加错误记录查询页面或接口。
- 按配置执行日志轮转。

## 测试要求

- 各稳定阶段均可暂停并恢复到原阶段。
- 处理中不能直接切换为 `PAUSED`。
- 恢复后计数器、轮次和历史记录不变。
- 每次状态和计数变化都有审计记录。
- 外部调用、重试和错误能够关联到会话与请求。
- 普通日志不包含密钥和完整敏感业务内容。
- 日志大小与备份数量从配置读取。

## 完成标准

- 任一测试会话的状态变化、模型调用、支持计数和失败原因都可以追溯。
- 暂停后的会话可以继续原流程。
- 本阶段自动测试全部通过。

## 合理性批判与不足

1. 限制暂停阶段显著降低 Demo 并发复杂度，但不能覆盖生产环境的请求取消需求。
2. 完整审计会增加数据量，当前方案没有定义长期保留和清理规则。
