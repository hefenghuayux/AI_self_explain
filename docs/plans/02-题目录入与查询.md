# 阶段 02：题目录入与查询

## 目标

完成 AI 评价所需题目材料的人工录入、编辑和读取。

## 范围

- `Question` 数据模型和数据库迁移。
- 新增、查看和编辑题目。
- 题目列表、详情及表单页面。
- 题目输入校验。

本阶段不实现录题人员权限，不调用 AI 自动生成任何题目材料。

## 修改内容

- 保存题目内容、标准答案、评分点、常见错误、其他解法、分层提示和完整解析。
- 为数组字段提供明确的录入与展示方式。
- 校验所有必填材料、空白项和重复评分点。
- 提供题目新增、详情、编辑接口和页面。

评分点保持自然语言文本，不增加额外业务判断或评分点 ID。

## 测试要求

- 合法题目可以保存并完整读取。
- 必填材料缺失时拒绝保存并显示准确错误。
- 空评分点、重复评分点和空提示被拒绝。
- 编辑后数据正确更新，未编辑字段保持不变。
- 数据库迁移可在空数据库执行。

## 完成标准

- 可以通过页面录入一题。
- API 和页面可以完整读取、编辑该题评价材料。
- 本阶段自动测试全部通过。

## 合理性批判与不足

1. 人工录题有利于保证 Demo 量规质量，但不能验证题目资产规模化生产成本。
2. 自然语言评分点方便录入，但后续结构化输出必须严格返回原文。
