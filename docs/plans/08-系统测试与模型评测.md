# 阶段 08：系统测试与模型评测

## 目标

形成可重复运行的 Demo 验收流程和离线模型评价能力。

## 范围

- 完整单元和集成测试。
- 关键浏览器流程测试。
- 人工真值导入。
- 三项模型指标。
- 逐样本与汇总评测报告。
- 启动、测试和验收说明。

本阶段不实现人工审核界面，也不在程序中固化指标通过阈值、最低样本量或审核分歧规则。

## 修改内容

- 定义固定版本的评测样本格式。
- 增加真值导入和离线评测命令。
- 支持读取历史评价或按指定模型重新评价。
- 计算错误放行率、正确误判率和正确但不完整识别率。
- 输出样本数、逐样本结果、模型版本、提示词版本、结构失败次数和 `NEED_HUMAN` 次数。
- 完善环境配置、启动、测试和验收说明。

## 测试要求

- 全部单元测试和集成测试通过。
- 文本与语音主流程均通过。
- 每个业务终态至少有一个完整测试场景。
- 三项指标使用固定样本可重复计算。
- 同一历史评价输入得到相同汇总结果。
- 评测报告包含模型、提示词和样本版本。
- 检查 Git 状态和 Git diff，不包含无关改动。

## 完成标准

- 新环境按说明配置后可以运行 Demo。
- 可以执行全部测试并生成离线评测报告。
- 报告能够追溯到逐样本真值与模型评价。
- Demo 具备按当前需求进行人工验收的条件。

## 合理性批判与不足

1. 离线指标依赖人工真值质量，样本偏差会直接影响结论。
2. 三项指标聚焦评价准确性，尚不能衡量提示是否真正促进学生进步。
3. 测试通过只能证明已覆盖场景符合规则，不能证明模型对开放表达具有普遍可靠性。
