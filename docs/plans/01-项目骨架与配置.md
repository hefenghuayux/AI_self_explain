# 阶段 01：项目骨架与配置

## 目标

按已确认的前后端分离文件架构，建立可启动、可测试且配置错误清晰可见的最小应用。

## 范围

- FastAPI 项目骨架。
- Vue 3、Vite、TypeScript 和 Element Plus 前端骨架。
- 前后端独立启动及 Vite `/api` 开发代理。
- Pydantic 配置模型。
- `.env.example`。
- SQLite 连接和 Alembic 初始化。
- 结构化日志。
- 健康检查接口和前端页面。

本阶段不实现题目录入、学生会话、AI 或 ASR 调用。

## 修改内容

- 按主技术方案建立 `backend/`、`frontend/`、`data/`、`scripts/` 及其规定的子目录。
- 在 `backend/app/` 下建立 `api/`、`core/`、`models/`、`schemas/`、`repositories/`、`services/`、`rules/` 和 `prompts/`。
- 在 `frontend/src/` 下建立 `api/`、`components/`、`views/`、`state/`、`types/` 和 `router/`。
- 声明 Python 与前端依赖和开发依赖。
- 增加应用入口、配置加载、数据库会话和日志初始化。
- 建立 Vue 3 应用入口、TypeScript 类型检查、Vite 构建配置和 Element Plus 初始化。
- 增加 FastAPI 健康检查接口及调用该接口的最小前端页面。
- 配置 Vite 将 `/api` 请求代理到 FastAPI；前端代码不写死后端地址。
- 配置 `.gitignore`，排除 `.env`、音频、SQLite 文件和构建产物。
- 为所有次数、阈值、超时、重试和路径建立配置字段。

配置缺失、类型错误或配置间约束不成立时停止启动并打印准确错误，不增加运行时兜底配置。

## 测试要求

- 应用可以使用一条命令启动。
- 健康检查接口和页面返回成功。
- TypeScript 类型检查通过。
- Vite 生产构建通过。
- Element Plus 基础组件可以正常渲染。
- 前端通过 `/api` 代理访问 FastAPI 健康检查接口。
- 后端停止时，前端明确显示接口调用错误。
- 测试数据库可以连接。
- 缺少必填配置时启动失败并指出字段。
- 非法次数、阈值、超时、退避数组和路径配置被拒绝。
- 日志中不输出密钥。

## 完成标准

- 新环境按说明安装依赖后可以启动服务。
- 浏览器能够访问健康检查页面。
- 前端可以调用 FastAPI 健康检查接口。
- 实际目录与主技术方案规定的文件架构一致。
- 前端没有状态转换、支持计数或阈值判断实现。
- 本地音频、SQLite 数据库和密钥配置不会被 Git 跟踪。
- 本阶段自动测试全部通过。
- Git diff 中只包含本阶段文件。

## 合理性批判与不足

1. 本阶段不能验证业务价值，但能尽早固定配置边界和运行方式。
2. 一开始引入 Alembic 会增加少量结构，但可以避免后续手工修改数据库。
3. 独立前端工程提高了多轮交互的可维护性，也增加了依赖安装、构建和接口联调成本。
4. 预先建立全部职责目录能够约束后续实现，但部分目录在早期阶段可能暂时为空，不应因此提前加入冗余封装。
